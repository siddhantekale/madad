# Madad

## Planned — Agents, Knowledge Store & Documents (design, not yet built) -> needs to be implemented still

Personal ChatGPT/Claude-style mobile chat app with an on-device, embedding-backed
knowledge store. Expo SDK 54 · React Native 0.81 · TypeScript · expo-router.

> Notes are written for an AI agent. Full design in [`ARCHITECTURE.md`](./ARCHITECTURE.md).

## Architecture (at a glance)

```
 ChatInput ─► SessionsContext ─► LLMProvider.send() ──► DeepSeek API (chat)
     │              │                                        │
     │              ▼                                        ▼
     │        AsyncStorage (sessions)                 assistant reply
     │                                                       │
     │                                          MessageBubble (Markdown + copy)
     │                                          + local notification if backgrounded
     ▼
 header "index" button ─► indexer ─► (summarize if long via LLM)
                                  ─► EmbeddingProvider.embed() ─► OpenAI text-embedding-3-small
                                  ─► expo-sqlite session_index (append-only Float32 BLOB)

 SettingsContext: provider/model + API keys (expo-secure-store)
 LLMProvider / EmbeddingProvider are swappable seams for a future custom endpoint.
```

Routes: `src/app/_layout.tsx` (Drawer), `index.tsx` (chat), `settings.tsx`. Source root is `src/`.

## Prerequisites

- Node 20/22/24 (repo currently runs on 23 with an Expo "unsupported engine" warning — works, off-range).
- Xcode + CocoaPods for native/device builds. macOS only.
- API keys entered **in-app** (Settings), not env: **DeepSeek** (chat) and **OpenAI** (embeddings).

```bash
npm install
```

## iOS

### Dev (fastest iteration)
```bash
npx expo start          # press i for simulator, or scan QR with Expo Go
```
Expo Go must match the project SDK (54). If Expo Go reports an incompatible SDK, use a build below instead.

### Standalone on a physical device (free Apple ID, ~7-day signing)
```bash
npx expo run:ios --device --configuration Release
```
- `Release` embeds the JS bundle → runs offline, no Metro, no Expo Go.
- `ios.bundleIdentifier` = `com.siddhantekale.madad`. Device needs Developer Mode on + unlocked.
- First time only: set the signing Team once via the Xcode GUI (`open ios/Madad.xcworkspace` → target → Signing & Capabilities → automatic + your Personal Team), then the CLI build works. On-device first launch: Settings → General → VPN & Device Management → Trust the developer cert.

### Build gotchas (important for agents)
- `/ios` and `/android` are git-ignored and regenerated by `npx expo prebuild`.
- After adding/removing **any native module**, do a clean native regen or the link fails with
  `Undefined symbols ... facebook::react::Sealable` (New-Arch Fabric codegen desync):
  ```bash
  rm -rf ios/Pods ios/build ios/Podfile.lock ~/Library/Developer/Xcode/DerivedData/Madad-*
  npx pod-install
  ```
- `Could not compute dependency graph ... unable to initiate PIF transfer session` → Xcode GUI is
  open and contending with the CLI build. Quit Xcode (or build only in Xcode), then retry; if wedged,
  `killall XCBBuildService`.
- Verify a native build without a device/signing:
  ```bash
  xcodebuild -workspace ios/Madad.xcworkspace -scheme Madad -configuration Debug \
    -sdk iphonesimulator -destination 'generic/platform=iOS Simulator' CODE_SIGNING_ALLOWED=NO build
  ```
